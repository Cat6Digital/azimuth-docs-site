Getting Help

IMPORTANT - Cat6 Digital cannot troubleshoot or support errors stemming from AI Models themselves. Provider timeouts, failed responses, etc., can and will happen from time to time due to many factors, and are entirely the domain of the respective AI model providers.

Contact:

Reporting Issues​

Before reporting:

  1. Check this guide's Troubleshooting section
  2. Check the FAQ
  3. Review Output Log (Window > Developer Tools > Output Log) for error messages

What to include in your bug report:

Required information:

  • UE version: (e.g., 5.7.1)
  • Orion version: (check plugin version in Edit > Plugins > Azimuth Orion)
  • Selected provider: (xAI, OpenAI, Anthropic, Google)
  • Selected model: (e.g., grok-code-fast-1, GPT-4o)

Issue details:

  • Steps to reproduce: (1. Open Blueprint, 2. Click Send, 3. Error appears)
  • Expected behavior: ("AI should respond with analysis")
  • Actual behavior: ("Error message")
  • Error message or unexpected behavior: (copy exact text)

Output Log excerpt:

  • Window > Developer Tools > Output Log
  • Copy last 20-50 lines (especially any lines with "Orion" or "Error")*
  • Paste into bug report *Cat6 Digital cannot troubleshoot non-Orion or general project errors

Screenshot (if applicable):

  • Screenshot of error dialog, chat window, or settings

Example bug report:

Subject: Blueprint screenshot fails to capture

Environment:

  • UE 5.7.1
  • Orion plugin version 1.0.2
  • Model: grok-4
  • Provider: xAI

Issue: When I click Send Screenshot to Orion from the Blueprint toolbar, the screenshot is not captured. The Output Log shows an Orion error.

Steps to reproduce:

  1. Open BP_GameMode in editor
  2. Click Orion toolbar: Send Screenshot to Orion
  3. Screenshot indicator does not appear in Orion status bar
  4. Output Log shows: LogOrionCapture: No window found matching title substring: 'Blueprint`

Expected: Screenshot attaches to Orion chat window and sends with next message.

Actual: Screenshot is not captured; error appears in the log.

Output Log (last 10 lines):

LogOrionCapture: User requested Blueprint graph capture
LogOrionCapture: Warning: No window found matching title substring: 'Blueprint'

Question: Why can’t Orion find the Blueprint window for capture? Does it depend on window focus or naming?

Feature Requests​

Submit feature requests via:

  • Email: support@cat6digital.com (subject: "Feature Request: [brief description]")

What to include:

  • Feature description: What do you want Orion to do?
  • Use case: How would this improve your workflow?
  • Examples/mockups: (if applicable) Screenshots, diagrams, or examples of the feature

Example feature request:

Feature: Orion haiku mode

Description: Optional mode that, when enabled, adds a short haiku at the end of each Orion response summarizing the Blueprint or answer.

Use case: When a long analysis is overwhelming, a quick haiku at the end gives a memorable summary—and makes reading logs a bit more fun.

Benefit: Fun, memorable summaries; novelty factor.
